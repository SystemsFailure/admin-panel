Возможности пользователя
 - создать аккаунт ( основанно на jwt токенах )
 - авторизироваться
 - заполнить данные о себе
 - создовать, редактировать, удалять, получать сообщения
 - обманиваится различными форматами файлов ( cloud storage )
 - активация начала выаолнения рабочего процесса
 - возможность брать на себя задачу из спика предложенного админом( главным менеджером ) компании, в которую приписан данный пользователь( работник )
 - состояние онлайн или оффлайн
 - откликаться на начало смены, созданной менеджером компании

Возможности админа( менеджера )
 - создать аккаунт ( основанно на jwt токенах )
 - авторизироваться с помощью специального токена
 - создовать компанию ( заполнив все ее данные )
 - добовлять, удалять в список работников в данную компанию
 - создовать пользователя( работника )
 - создовать, редактировать, удалять, получать сообщения
 - обманиваится различными форматами файлов ( cloud storage )
 - создовать tasks и отправлять на выполнение работникам
 - следить за процессом выполнения задачи
 - открывть, закрывать смену
 - выстовлять крайний срок выполнения задач, который будут видны всем сотрудникам компании ( проекта )
 - связать компанию с проектами и сотрудниками


Аналитические расчеты
 - возможность просмотреть среднее значение продуктивности ( на основе закрытых задач проекта ( компании ) и времени потраченного на них )
 - возможность оценить качество выполненых задач
 - возможность оценить условия труда ( на основе международных соглашений об уловиях труда и данных получаемых в ходе работы сотрудников )
 - различные манипуляции с пользователями( работниками ) такие как - получения уникальных сотрудников ( на основе имени, должности и т.д. ), средняя зп всех сотрудников, среднее выполнение задач, худшие и лучшие задачи, простые/сложные задачи и т.д.


Сущности
 - Users
 - Admins( Manager )
 - Files
 - Messages
 - Tasks
 - Companys
 - Projects
 - Shifts ( Working shift )


- На каждый маршрут ставиться роль и Guard маршрута
- Http ошибки на маршрутах отробатываються с помощью Filters
- Pipes( встроенные в nest ) проверяют данные запроса
- Токены активации передаются через cookies
- Refresh токены через Sessions
- пароли хэшируються с помошью алгоритма SHA256 => уходят на сервер, там шифруются через алгоритм AES => Его ключи шифруются через RSA и передаются в зашифрованном виде на клиент
